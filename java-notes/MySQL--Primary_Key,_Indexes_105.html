Primary Key, Indexes


Primary Key (Первичный ключ) - это поле, которое позволяет однозначно идентифицировать запись в таблице

Позволяет различать пользователей друг от друга если их данные полностью совпадают.

Значение первичного ключа должно быть уникально для всех записей

Уникальный индекс

SCHEMAS / employees / Tables / employees / Пр.кн. / Alter Table / Пометить столбец:

UQ = Unique index

В одной и той же таблице может быть несколько уникальных ключей

Уникальный индекс может содержать несколько колонок, т.е. нет ограничения на кол-во полей, которые должны входить в уникальный индекс:

SCHEMAS / employees / Tables / employees / Пр.кн. / Alter Table / Indexes:

• Выбрать уникальный ключ

• Добавить сюда какую-то другую колонку, тогда уникальность будет учитываться в разрезе двух полей

• Либо создать новый уникальный индекс и выбрать нужные поля

Обыкновенные регулярные индексы - могут быть задействованы для оптимизации и ускорения запросов

USE employees;

SELECT * FROM employees WHERE hire_date = '1986-06-26';

CREATE INDEX hire_date ON employees (hire_date); // MySQL вывело для себя все уникальные значения по колонке hire_date и растасовало ссылки на работников по секциям; теперь при запросе по hire_date = '1986-06-26' MySQL будет сканировать не всю таблицу, а только ту часть, которая соответствует данному критерию

SELECT * FROM employees WHERE hire_date = '1986-06-26'; // Время запроса значительно уменьшится после создания индекса

Составные индексы (индексы, состоящие из нескольких полей)

SELECT * FROM employees WHERE hire_date = '1986-06-26' AND gender = 'F';

CREATE INDEX gender_hire_date ON employees (gender, hire_date);

SELECT * FROM employees WHERE hire_date = '1986-06-26' AND gender = 'F';

Не стоит значительно злоупотреблять индексами, т.к. каждый созданный индекс сохраняется на дисковом пространстве. И при запросе MySQL сначала выбирает нужный индекс, а потом выполняет запрос.
