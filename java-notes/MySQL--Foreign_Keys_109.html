Foreign Keys


Внешние ключи и целостность данных

Внешний ключ - ссылка на первичный ключ в другой таблице

Внешними ключами могут стать те поля, которые играют роль ссылок одной таблицы на другую

Посмотреть какие внешние ключи есть у таблицы:

• film_category / Foreign Keys

или

• film_category / Пр.кн. / Alter Table / Foreign Keys

Foreign Key Name - имя внешнего ключа

Referenced Table - таблица B, на которую он ссылается

Column - поля текущей таблицы A, которые входят во внешний ключ

Referenced Column - поля Column таблицы A будут ссылаться на колонку таблицы B

Внешние ключи обеспечивают целостность данных за счет стратегий (Foreign Keys Options) на удаление и обновление:

• RESTRICT - самая строгая стратегия поведения внешних ключей. Если мы пытаемся обновить идентификатор какой-то записи или удалить строку, на которую ссылаются данные в других таблицах, то MySQL не позволит это сделать выбросив ошибку

• CASCADE - каскадно обновляет или каскадно удаляет все связующие данные - это очень опасная стратегия, которую нужно применять осознанно. Если обновить первичный ключ какого-то фильма, то он автоматически каскадно обновится во всех ссылочных полях во всех таблицах, где проставлена эта стартегия

• SET NULL - выставить значение NULL в те ссылки, записи на которые удалились

• NO ACTION - отсутствие какого-либо действия
