Git


git init - инициализация git-репозитория

git status

git add 1.txt - добавить файл в индекс

git add 1.txt 2.txt

git add ./Lesson02/*

git add *

git commit - фиксация

git commit -m “Added file 1.txt”

gitk --all - графическое отображение дерева (--all - отображение всех веток)

git config --list

git config --local --list - локально (в текущей папке)

git config --global --list - в глобальных настройках Git

git config --local user.name “Vasya”

git branch - посмотреть ветки

git branch 34-create - создать ветку “34-create”

git checkout 34-create - перейти на четку “34-create”

git reset --hard HEAD^ - (HEAD - указатель на текущий комит (текущее состояние, на котором сейчас находимся))

(^ - один комит назад)

(^^ - два комита назад)

--hard - после команды изменится и Git, и файловая система

git reset --hard 321fd3847 - переход на комит, хэш которого равен “321fd3847”, можно указать несколько первых символов

git reset --soft HEAD^ - файловая система не изменяется, изменяется только внутренняя структура Git

git rebase master - актуализирование текущей ветки к ветке master (меняется основание ветки) С помощью команды rebase вы можете взять все коммиты из одной ветки и в том же порядке применить их к другой ветке.

git merge master - слияние веток, при этом создается дополнительный комит

git cherry-pick

git push -f - force push (не делать)

“.gitignore” - файл, который должен лежать в корне (перечисление того, что не должно попасть в git-репозиторий)

*.zip - все zip-файлы

*.jar

*.class

.idea - папка “.idea”

#Git проверяет также вложенные папки

# - комментарий

“.gitattribures”

*.png binary - для этих файлов не будут делаться DIFF-ы, файлы будут заменены полностью

* text-auto

*.txt text

*.bat text eol-crlf - указание символов конца строки для Windows (bat - скриптовый виндовый файл)

*.sh text eol-lf - указание символов конца строки для Линукс

*.jpg -text аналог binary

https://git-scm.com/docs/gitattributes

nano 1.txt - создание текстового файла в редакторе nano

cat 1.txt - вывести содержимое файла в командной строке Git Bash

echo sdf > 123.txt - создать файл 123.txt с содержимым “sdf”

ll - это ls с параметром -l "ls -l" - вывод списка файлов в строку (линукс команда)

ls - вывод списка файлов столбцом (линукс команда)

ls -lah - показать скрытые файлы (a), красивок представление размера файлов (h)

rm 1.txt - удалить файл “1.txt”

~ - рабочая папка

ls -lah ~ содержимое рабочей папки

ls -lah ~/.ssh содержимое папки “.ssh” в рабочей папке

git remote -v - вывести название удаленного репозитория (в IDEA / Terminal)

git remote update - лучше делать вместо git pull
