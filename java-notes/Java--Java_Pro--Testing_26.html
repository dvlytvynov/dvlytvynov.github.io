Testing


Unit tests - юнитом считается одна функция, один метод, которая показывает зависимость чего-то от чего-то. Тесты нужно писать для каждой нетривиальной функции или метода.

Integration tests - интеграционное тестирование - часть программы запускается с другой частью программы и проверяется взаимодействие этих частей. Создаем экземпляр класса или классов, в конструктор передаем набор каких-то данных. Чаще всего эти данные будут не настоящими, а так называемыми моками. И проверяем правильно ли работает этот класс.

Нужно писать тест так, чтобы внутри теста мы попадали во все ветки (if, switch, циклы). При проверке if(условие1 или условие2) нужно выполнить тест, который попадет в условие1, потом условие2, потом в оба условия.

Подход ААА - arrange(настраиваем все), act(производим вызов нужных функций, т.е. выполняем тест), assert(проверяем, что все вернулось правильно)

int[] arr = new int[]{3, 4, 5, 8, 1, 2}; // arrange

int[] output = sort(arr); // act

assert arr == output : "Error ....... ....."; // assert

assert output[0] != 3 : "Error ....... ....."; // assert

assert output[1] != 4 : "Error ....... ....."; // assert

assert output[2] != 5 : "Error ....... ....."; // assert
