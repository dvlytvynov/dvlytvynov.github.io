Java Style (remarks)


Remarks (Java Elementary course)

• Одно ДЗ д.б. в одном мердж-реквесте

• Константы должны быть выше члена экземпляра

• Не оставлять пустые строки там где они не нужны

• Лучше, чтобы каждый тест не влиял на результат других

• Абсолютные пути - плохая практика

• Использовать printf (чтобы не использовать “+”)

• Если у вас в коде появляются классы с названием Helper (вспомагательный класс), то у вас что-то не так с архитектурой

• Если класс А ни от кого не наследуется, значит другие классы будут зависеть от этой имплементации класса А (это нарушает букву D в SOLID (код должен зависеть от абстракций, а не от реализации)). Реализацию передавать по интерфейсной ссылке, а не по экземплярной

• Классы разделять по пакетам

• Файлы, которые не относятся к коду, должны находиться в папке ресурсов (resources), а не в папке исходных кодов (src)

• Слова в названии пакета лучше разделять “.”

• main можно оставить вне пакета

• Множественное число в названии класса - плохо

• Порядок размещения полей в классе:

1. Статические константы.

2. Статические переменные.

3. Нестатические константы.

4. Нестатические переменные.

• В названиях методов, классов и т.д. не жалеть букв - Java многословна

• Метод должен помещаться на одном экране

• Название метода должно быть понятным (понятно, что метод делает)

• Если вызываются методы с одинаковым интерфейсом и меняется только входная сторка, лучше вместо switch сделать мапу с лямбдами

• Лучше написать несколько маленьких методов, чем один со сложной логикой (сложно будет тестировать)

• Методы, возвращающие boolean (предикаты) должны начинаться с is или has, например isDataCorrect()

• Вместо метода getDetailInfo() лучше переопределить метод toString(). А из toString() вызывать getDetailInfo().

• list.sort(null) - плохая сортировка. Нужно использовать:

Collections.sort(new ArrayList<String>(), (o1, o2) -> o1.compareTo(o2));

Collections.sort(new ArrayList<User>(), (o1,o2)->{
            if(o1.getAge()>a2.getAge())
                return 1;
            if(o1.getAge()<a2.getAge())
                return -1;
            return 0;
        });

Или в классе задать implements Comparable

public class User implements Comparable<User>{}

Тогда при сортировке не нужно указывать компаратор

Collections.sort(new ArrayList<User>());

Лучше использовать TimeUnit.SECONDS.sleep(1) вместо Thread.sleep(1000);
