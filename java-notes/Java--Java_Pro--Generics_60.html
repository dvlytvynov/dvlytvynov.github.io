Generics


Generics (Обобщения)

Обобщения существуют до момента компиляции. Во время компиляции вместо T, V, <T>.... подставляются конкретные Object.

Нельзя создать объект типа T : ob = new T();

Нельзя создать статические переменные типа T : static T ob

В статическом методе нельзя использовать параметр типа T : static T ob() {...} - так как нельзя создать экземпляр статического класса.

Нельзя создать массив типа T (т.к. нужно делать грязное преобразование T[] arr = (T[]) new Object[10] )

List<?> - параметризовано чем-угодно. Если нужно параметризовать, но заранее не знаем, чем будет параметризовано.

public class SomeClass<T extends OtherClass> {...} - (при объявлении класса) класс можно параметризовать типом OtherClass и его наследниками.

SomeClass<? super OtherClass> some = new SomeClass<>(); - (при создании экземпляра) классSomeClass параметризуем классом OtherClass или его наследниками.

List<? extends Number> - список будет хранить только числа. Список параметризован чем угодно, но это что угодно должно расширять Number.

List<V extends T> - V - любой наследник от типа, которым является T.

List<? super Integer> - ограничение с верхней стороны, ограничение дает возможность добавлять в список только Integer.

class TestClass<T extends OtherClass & FirstInterface & SecondInterface …> { ... - ограничение включает тип класса и тип одного или более интерфейсов.

instanceof - оператор проверки принадлежности типа

instanceof расшифровывает понятие «объект создан на основе класса»

b instanceof ClassB = true - если экземпляр b создан на основании класса ClassB (b - экземпляр класса ClassB).

b instanceof ClassA = true - если экземпляр b создан на основании класса ClassA (если ClassB - наследник ClassA).
