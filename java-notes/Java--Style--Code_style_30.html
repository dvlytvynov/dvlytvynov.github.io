Code style


Remarks (Java Enterprise course)

1. Максимум одна пустая строка между франментами кода (много пустых строк - плохо)

2. Форматирование кода. В IDEA: Ctrl + Alt + L (Code/Reformat code)

3. Использовать относительные пути: String path = “./Lesson-02/src/main/java/hillel/course/1.txt”;

4. Одна пустая строка в конце файла

5. Название главного класса (который отвечает за запуск) = “App......”, например, AppHibernate.

6. Главный класс должен быть простой - должен только запускать.

7. Последовательность в классе:

1) Статические переменные

2) Нестатические переменные

3) Конструктор

4) Публичные методы

5) Приватные методы

8. Рекомендуется

1) минимализм, не усложнять код, так как больше кода - может быть больше ошибок, больше времени требуется на проверку. На проверку (на ревью) 300 строк кода требуется около 1 часаработы программиста.

2) Упростить ввод параметров (см. HomeTask-04) - каждый параметр вводить отдельно, чтобы пользователю было понятно.

9. Рекомендуется хранить пароли или в переменных окружения или в дженкинсе.

10. Логгирование тестов - уровень DEBUG.
